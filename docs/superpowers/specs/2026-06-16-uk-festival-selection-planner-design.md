# UK节日选品备货决策工具 — 设计文档

**日期**：2026-06-16
**状态**：已定稿，待实现
**业务背景**：亚马逊UK电商团队，3个店铺，铺货型FBA，客单价 £5.99-9.99，1688拿货，每月上新200-300 SKU，前期每款20件测款。

---

## 1. 目标与定位

### 核心问题
为节日款/活动款/趋势款，**提前2个月**完成选品与备货决策。需要：
1. 知道英国各个大小节日/活动节点
2. 知道每个节点卖什么货
3. 知道何时启动备货、各阶段做什么

### 交付物定位
**选品决策工具（行动导向）**——看完知道"该卖什么、何时备货、备多少"。
不是知识库（学习导向），也不是纯流程看板（节奏导向）。

### 成功标准
- 打开HTML，5秒内知道"今天最该处理哪个节日"
- 每个节日有具体可执行的SKU建议（非泛泛而谈）
- 备货时间线可视化，倒推到"选品截止日"
- 进度可追踪，本地保存，不丢失

## 2. 范围

### 时间范围
**未来6个月滚动**：2026年7月 - 12月（正好覆盖提前2个月备货窗口，包含万圣节/黑五/圣诞三大S级节点）。

### 品类范围
4大品类（均已确认覆盖）：
- 🎃 节日装饰与派对用品
- 🎁 礼品/赠送小件
- 👕 服饰配件/装饰配饰
- 🏠 节日主题家居/日用品

### 店铺维度
**统一规划**——按节日维度追踪，备注字段标注店铺分布（如"A店上架5款/B店3款"）。不做按店铺独立追踪（避免数据×3冗余）。

### 不包含（YAGNI）
- 实时销量数据接入（无API、静态文件无法维护）
- 跨年规划（2027年情人节等，趋势数据预测性差）
- 按店铺独立追踪
- 用户登录/云端同步（单一自包含文件）

## 3. 信息架构与页面结构

单页垂直滚动，自上而下5个区块：

### ① 顶部固定导航栏（Sticky Header）
- 标题/Logo
- 月份锚点导航（7月~12月）
- 今日日期显示

### ② 紧急度看板（Hero Dashboard）——决策入口
- 距下一个需备货节日的倒计时
- 4个统计卡片：
  - 🔴 紧急（T-55已过未选品）
  - 🟠 本周启动（T-55距今≤7天）
  - 🟡 本月备货（T-55距今≤30天）
  - 🟢 规划中（T-55距今>30天）
- 统计卡片可点击 → 快速筛选对应紧急度的节日
- 倒计时时间轴：未来6个月节点可视化条

### ③ 筛选工具栏（Sticky Filter Bar）
组合筛选（AND逻辑）：
- 按品类：全部 / 装饰 / 礼品 / 服饰 / 家居
- 按月份：7-12月 / 全部
- 按紧急度：全部 / 紧急 / 本周 / 本月 / 规划
- 按状态：全部 / 未启动 / 选品中 / 已下单 / 已到仓 / 已上架
- 搜索框：模糊匹配节日名/SKU/关键词
- 重置按钮

### ④ 节日卡片列表（主体，按月份分组）
按月份分组渲染，每张卡片承载一个节日的全部决策信息（详见§6卡片结构）。
- S级卡片默认展开
- A/B级卡片默认折叠，点击展开

### ⑤ 页脚
- 使用说明
- 数据免责声明（"选品建议基于经验推荐，售价/成本为区间参考，请结合实时数据验证"）
- JSON导入/导出备份按钮
- 清除所有进度按钮（带二次确认）

## 5. 选品建议结构

### SKU数据结构

```js
product = {
  sku: "南瓜LED串灯",                      // 中文商品名
  skuEn: "Pumpkin LED String Lights",       // 英文Listing参考词
  category: "decor",                        // decor|gift|apparel|home
  costRange: "¥8-15",                       // 1688拿货成本区间（人民币）
  priceRange: "£6.99-8.99",                 // 建议售价区间（英镑）
  margin: "约45-55%",                       // 预估毛利率（扣FBA费后）
  matchScore: 4,                            // 匹配度1-5星
  riskLevel: "低",                          // 风险评级：低|中|高
  riskNote: "注意LED的UKCA/CE认证；避免Disney南瓜车IP",
  keywords: ["halloween lights","pumpkin decor","outdoor halloween"],
  sourcing: "1688搜索词: 万圣节南瓜灯串",
  notes: ""
}
```

### 字段说明

| 字段 | 取值规则 |
|---|---|
| `costRange` | 区间值，反映1688价格波动。不写精确值 |
| `priceRange` | 区间值，落在 £5.99-9.99 客单价带内 |
| `margin` | 扣除FBA配送费、佣金后的预估毛利率 |
| `matchScore` | 1-5星。综合评估：客单价契合/轻小件/FBA费用/退货率 |
| `riskLevel` | 低/中/高。见下方风险评级规则 |
| `riskNote` | 具体风险说明：IP/认证/退货/情绪敏感 |

### 风险评级规则
| 等级 | 适用场景 |
|---|---|
| **低** | 常规品类，无IP/认证/退货风险 |
| **中** | 需注意认证（电子产品UKCA）、轻微季节滞销 |
| **高** | 涉IP侵权可能（动漫/影视）、高退货率（服饰尺码）、情绪敏感（Remembrance Day） |

### 卡片内选品清单展示
按4品类分组，每行一个SKU，列：SKU名 / 拿货成本 / 售价 / 毛利率 / 匹配度 / 风险。
支持品类标签筛选（卡片内）。

## 6. 备货时间线（双段物流）

### 核心策略
**空运首批50件测款 + 卡航大货同时下单**——铺货型FBA典型打法。
- 空运首发：快速到仓测款，验证转化
- 卡航大货：成本可控，覆盖主销量
- 大货与首批同时下单（不等测款结果，否则卡航周期来不及）

### 物流方式配置

```js
logisticsModes = {
  air:    { leadTime: 15, production: 7,  transit: 8,  label: "空运" },
  truck:  { leadTime: 35, production: 10, transit: 25, label: "卡航/快铁（默认主力）" },
  sea:    { leadTime: 55, production: 15, transit: 40, label: "海运" }
}
```

| 物流方式 | 总周期 | 生产 | 头程运输 | 适用 |
|---|---|---|---|---|
| ✈️ 空运 | 15天 | 7天 | 8天 | 首批测款/急补货 |
| 🚆 卡航/快铁 | 35天 | 10天 | 25天 | **主力（默认）** |
| 🚢 海运 | 55天 | 15天 | 40天 | 大货最低成本 |

### 双段流程里程碑（以卡航为主力）

总提前期约 **55天**（≈2个月，符合需求）。

**可勾选里程碑**（localStorage追踪状态）：

| 阶段 | tMinus | 动作 |
|---|---|---|
| **选品+下单** | T-55 | 定SKU；同时下空运首批50件 + 卡航大货订单 |
| **空运首批到仓** | T-47 | 空运8天到仓，开始测款 |
| **卡航大货发运** | T-25 | 大货出厂发UK |
| **大货到仓** | T-14 | FBA收货上架，Listing转Active |
| **节日** | T-0 | 广告加投，促销启动 |

**观察期（非勾选项，仅显示提醒）**：
- T-47 ~ T-30：测款观察期——观察首批转化/点击/自然单，决定是否追加空运补货

### 里程碑数据结构

```js
milestone = {
  id: "selection",
  name: "选品+下单",
  tMinus: 55,
  date: "2026-09-06",     // 自动计算：节日日期 - 55天
  status: "pending",      // pending|done（localStorage保存）
  actions: [
    "确定2-3个核心SKU",
    "1688询价3家供应商比价",
    "下空运首批50件",
    "下卡航大货订单",
    "核算FBA费用确认毛利率≥40%"
  ]
}
```

### 紧急度状态机
按"选品截止日（T-55）距今天数"判定：

| 状态 | 条件 | 颜色 | 含义 |
|---|---|---|---|
| 🔴 紧急 | T-55已过且选品未完成 | 红 | 已来不及，考虑空运救场或放弃 |
| 🟠 本周启动 | T-55距今 ≤ 7天 | 橙 | 本周必须选品 |
| 🟡 本月备货 | T-55距今 ≤ 30天 | 黄 | 本月启动 |
| 🟢 规划中 | T-55距今 > 30天 | 绿 | 观察 |
| ⚫ 已过 | 节日已过 | 灰 | 归档 |

### 物流方式切换
每个节日卡片顶部有物流方式切换器（海运/卡航/空运）。切换后：
- 该卡片的时间线日期实时重算
- 选品截止日重算
- 紧急度标签重算
- 仅影响显示，不影响其他节日

## 10. 技术实现

### 交付物
单一自包含HTML文件：`uk-festival-planner.html`
- 所有CSS内联在 `<style>` 标签
- 所有JS内联在 `<script>` 标签
- 所有节日数据作为JS对象内联
- 无外部依赖（无CDN、无字体加载、无图片外链）
- 双击即可在浏览器打开，可发微信/邮件

### 技术栈
- 纯原生 HTML + CSS + JavaScript（无框架）
- 响应式（桌面优先，移动端可用）
- localStorage 持久化

### 数据组织
- 节日数据作为JS对象数组（`FESTIVALS`常量）
- 每个节日的products、milestones、validation内嵌
- 渲染逻辑：遍历FESTIVALS → 生成DOM

## 12. 不确定项 / 后续迭代

- **数据准确性**：SKU建议、拿货成本、毛利率为经验估值，需用户结合实际验证。首版基于通用经验，后续可根据真实销售数据修正。
- **节日完整性**：23个节点已覆盖7-12月主要机会，若发现遗漏可追加（如Black Friday Week、Small Business Saturday等子节点）。
- **跨年扩展**：当前仅到2026年12月，2027年情人节/母亲节等需另起一份或滚动更新。
